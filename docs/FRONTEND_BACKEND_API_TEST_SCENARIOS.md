## 테스트 환경 설정

### 사전 준비사항

1. **백엔드 서버 실행**
   - 백엔드 서버가 `http://localhost:8080`에서 실행 중이어야 합니다
   - 데이터베이스가 정상적으로 연결되어 있어야 합니다

2. **프론트엔드 서버 실행**
   - 프론트엔드 개발 서버가 실행 중이어야 합니다 (일반적으로 `http://localhost:5173`)

3. **브라우저 개발자 도구**
   - Network 탭을 열어 API 호출을 모니터링할 수 있도록 준비합니다
   - Console 탭을 열어 에러 메시지를 확인할 수 있도록 준비합니다

## 랜딩페이지 테스트 시나리오

**페이지 URL**: `/`  
**관련 API**: 
- `GET /api/v1/promotions/current` (프로모션 정보 조회)
- `GET /api/v1/pre-registrations/check-email?email={email}` (이메일 중복 체크)

### 시나리오 1: 프로모션 정보 조회

**목적**: 랜딩페이지 로드 시 현재 프로모션 정보가 정상적으로 표시되는지 확인

**테스트 단계**:

1. 브라우저에서 `http://localhost:5173/` 접속
2. 개발자 도구 Network 탭 열기
3. 페이지 로드 확인
4. Network 탭에서 `GET /api/v1/promotions/current` 요청 확인
   - **예상 상태 코드**: `200 OK`
   - **예상 응답**: 프로모션 단계(A/B), 할인율, 기간 정보 포함
5. 페이지에서 프로모션 배너가 정상적으로 표시되는지 확인
   - Phase A인 경우: "🔥 30% 할인" 배지 표시
   - Phase B인 경우: "✨ 10% 할인" 배지 표시
   - 카운트다운 타이머가 정상적으로 작동하는지 확인

**확인 사항**:
- [ ] API 호출이 성공적으로 이루어졌는가?
- [ ] 프로모션 배너가 정상적으로 표시되는가?
- [ ] 할인율이 올바르게 표시되는가?
- [ ] 카운트다운 타이머가 정상 작동하는가?

## 회원가입 페이지 테스트 시나리오

**페이지 URL**: `/signup`  
**관련 API**:
- `POST /api/v1/auth/signup` (회원가입)
- `POST /api/v1/pre-registrations` (사전등록 - 유료 요금제 선택 시)
- `GET /api/v1/pre-registrations/check-email?email={email}` (이메일 중복 체크)

### 시나리오 1: 기본 요금제 회원가입 (이메일/비밀번호)

**목적**: 기본 요금제로 이메일/비밀번호 회원가입이 정상적으로 작동하는지 확인

**테스트 단계**:

1. 회원가입 페이지 접속 (`/signup?plan=기본`)
2. 개발자 도구 Network 탭 열기
3. 다음 정보 입력:
   - **이메일**: `test@example.com`
   - **이름**: `테스트 사용자`
   - **비밀번호**: `password123`
   - **비밀번호 확인**: `password123`
4. 약관 동의 체크박스 선택:
   - [필수] 이용약관 동의
   - [필수] 개인정보처리방침 동의
   - [선택] 마케팅 정보 수신 동의 (선택사항)
5. "가입 후 무료 데모 체험하기" 버튼 클릭
6. Network 탭에서 `POST /api/v1/auth/signup` 요청 확인
   - **예상 상태 코드**: `201 Created` 또는 `200 OK`
   - **예상 응답**: 사용자 정보 및 토큰 포함
7. `/writing-demo` 페이지로 리다이렉트되는지 확인

**확인 사항**:
- [ ] 회원가입 API 호출이 성공적으로 이루어졌는가?
- [ ] 응답에 사용자 정보와 토큰이 포함되어 있는가?
- [ ] 회원가입 후 사업계획서 작성 데모 페이지로 이동하는가?
- [ ] 사전등록 API는 호출되지 않는가? (기본 요금제이므로)

### 시나리오 2: 유료 요금제 회원가입 및 사전등록 (플러스/프로/프리미엄)

**목적**: 유료 요금제 선택 시 회원가입과 사전등록이 모두 정상적으로 작동하는지 확인

**테스트 단계**:

   - **이메일**: `premium@example.com`
   - **이름**: `프리미엄 사용자`
   - **비밀번호**: `password123`
   - **비밀번호 확인**: `password123`
   - **전화번호**: `010-1234-5678` (유료 요금제 필수)
   - **사업 분야**: 드롭다운에서 선택 (예: "IT/소프트웨어")
4. 약관 동의 체크박스 선택:
   - [필수] 이용약관 동의
   - [필수] 개인정보처리방침 동의
   - [선택] 마케팅 정보 수신 동의
5. "가입하고 {할인율}% 할인 받기" 버튼 클릭
6. Network 탭에서 다음 API 호출 확인:
   - `POST /api/v1/auth/signup` (회원가입)
     - **예상 상태 코드**: `201 Created` 또는 `200 OK`
   - `POST /api/v1/pre-registrations` (사전등록)
     - **예상 상태 코드**: `201 Created`

7. `/writing-demo` 페이지로 리다이렉트되는지 확인

### 시나리오 2: Wizard 데이터 저장 (1단계 - 문제 인식)

**목적**: Wizard 단계에서 입력한 데이터가 정상적으로 저장되는지 확인

**테스트 단계**:

1. `/wizard/1` 페이지 접속
2. 개발자 도구 Network 탭 열기
3. 다음 정보 입력:
   - **사업 아이템명**: `AI 학습 플랫폼`
   - **사업 아이템 요약**: `개인 맞춤형 AI 학습 솔루션`
   - **문제 인식**: `학습자마다 다른 학습 속도와 스타일을 고려하지 못하는 기존 교육 시스템의 한계`
   - **문제 증거**: `학습자 만족도 조사 결과 70%가 학습 속도 불일치 문제 호소`
   - **타겟 고객**: `20-30대 직장인 및 대학생`
4. "다음" 버튼 클릭
5. Network 탭에서 `PUT /api/v1/projects/{projectId}/wizard` 요청 확인
   - **예상 상태 코드**: `200 OK`
   - **예상 요청 본문**:
     ```json
     {
       "currentStep": 1,
       "stepData": {
         "item-name": "AI 학습 플랫폼",
         "item-summary": "개인 맞춤형 AI 학습 솔루션",
         "problem": "...",
         "problem-evidence": "...",
         "target-customer": "..."
       },
       "isStepComplete": true
     }
     ```
   - **예상 응답**: 저장 시간, 진행 상태 업데이트 포함
6. `/wizard/2` 페이지로 이동되는지 확인

**확인 사항**:
- [ ] Wizard 데이터 저장 API 호출이 성공적으로 이루어졌는가?
- [ ] 요청 본문에 입력한 데이터가 포함되어 있는가?

2. 개발자 도구 Network 탭 열기
3. 페이지 로드 확인
4. Network 탭에서 `GET /api/v1/evaluations/{evaluationId}/result` 요청 확인
   - **예상 상태 코드**: `200 OK`
   - **예상 응답**: 
     - 총점, 등급, 합격률
     - 영역별 점수 (M.A.K.E.R.S 6개 영역)
     - 강점 목록
     - 약점 목록
     - 개선 권장사항
5. 페이지에 평가 결과가 정상적으로 표시되는지 확인:
   - 총점 및 등급 표시
   - 영역별 점수 차트/표시
   - 강점/약점 목록 표시
   - 개선 권장사항 표시

## 에러 처리 테스트 시나리오

### 시나리오 1: 네트워크 오류 처리

**목적**: 백엔드 서버가 응답하지 않을 때 프론트엔드가 적절히 처리하는지 확인

**테스트 단계**:

1. 백엔드 서버 중지 (`http://localhost:8080` 접근 불가 상태)
2. 프론트엔드에서 API 호출이 필요한 페이지 접속 (예: `/writing-demo`)
3. 개발자 도구 Network 탭에서 API 호출 실패 확인
4. Console 탭에서 에러 메시지 확인
5. 페이지에 에러 메시지가 표시되는지 확인

**확인 사항**:
- [ ] 네트워크 오류 시 적절한 에러 메시지가 표시되는가?
- [ ] 사용자에게 명확한 안내가 제공되는가?
- [ ] 애플리케이션이 크래시되지 않는가?

### 시나리오 2: 인증 오류 처리

3. 인증이 필요한 API 호출 페이지 접속
4. 개발자 도구 Network 탭에서 `401 Unauthorized` 응답 확인
5. 로그인 페이지로 리다이렉트되거나 에러 메시지가 표시되는지 확인

**확인 사항**:

- [ ] 사용자에게 명확한 안내가 제공되는가?

### 시나리오 3: 유효성 검증 오류 처리

**목적**: 잘못된 입력 데이터로 API 호출 시 적절히 처리하는지 확인

**테스트 단계**:

5. 페이지에 유효성 검증 에러 메시지가 표시되는지 확인

**확인 사항**:

## 참고사항

1. **API 호출 모니터링**: 모든 테스트 시 개발자 도구 Network 탭을 열어 API 호출을 모니터링하세요.

3. **응답 데이터 확인**: Network 탭에서 각 API 응답의 `success`, `data`, `error` 필드를 확인하세요.

6. **데이터베이스 상태**: 테스트 전에 데이터베이스가 정상적으로 연결되어 있는지 확인하세요.
