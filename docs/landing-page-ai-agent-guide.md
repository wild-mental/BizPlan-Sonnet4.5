# 🤖 AI 에이전트 활용 랜딩페이지 보강 교육 가이드

> **교육 대상**: AI 코딩 에이전트와 협업하여 랜딩페이지를 개선하고자 하는 기획자/PM/마케터  
> **소요 시간**: 약 2시간  
> **난이도**: ⭐ 입문 (코딩 경험 불필요)  
> **핵심 역량**: 명확한 요구사항 전달, AI 에이전트와의 효과적인 대화

---

## 🎯 이 교육의 목표

코드를 직접 작성하지 않고, **AI 에이전트에게 올바른 지시를 내려** 프리미엄 랜딩페이지를 완성하는 방법을 학습합니다.

### 당신이 할 일
✅ 무엇을 원하는지 명확하게 설명하기  
✅ 콘텐츠와 카피라이팅 준비하기  
✅ AI의 결과물을 검토하고 피드백하기  
✅ 반복적으로 개선 요청하기

### AI 에이전트가 할 일
🤖 코드 작성 및 수정  
🤖 기술적 구현  
🤖 버그 수정  
🤖 스타일링 적용

## 1. AI 에이전트와 효과적으로 소통하는 법

### 1.1 좋은 프롬프트의 구조

```
┌─────────────────────────────────────────┐
│  🎯 좋은 프롬프트 = WHAT + WHERE + HOW  │
├─────────────────────────────────────────┤
│                                         │
│  WHAT: 무엇을 원하는가?                 │
│  • 구체적인 기능/UI 설명                │
│  • 원하는 결과물의 모습                 │
│                                         │
│  WHERE: 어디에 적용할 것인가?           │
│  • 특정 섹션/파일 지정                  │
│  • 위치 관계 명시 (위/아래/앞/뒤)       │
│                                         │
│  HOW: 어떤 스타일/방식으로?             │
│  • 색상, 레이아웃, 애니메이션           │
│  • 참고할 기존 요소                     │
│                                         │
└─────────────────────────────────────────┘
```

### 1.2 ❌ 나쁜 프롬프트 vs ✅ 좋은 프롬프트

| ❌ 나쁜 프롬프트 | ✅ 좋은 프롬프트 |
|-----------------|-----------------|
| "히어로 섹션 만들어줘" | "현재 히어로 섹션 위에 새로운 히어로 섹션을 추가해줘. 헤드라인은 '정부지원금 합격 사업계획서, 10분이면 충분합니다'로 해줘" |
| "네비게이션 추가해" | "최상단에 고정 헤더 네비게이션을 넣고 싶어. 왼쪽에는 로고, 중앙에는 섹션 이동 링크, 오른쪽에는 CTA 버튼을 배치해줘" |
| "예쁘게 해줘" | "글래스모피즘 스타일로 반투명 배경에 블러 효과를 적용해줘" |
| "안 돼" | "드래그할 때 텍스트가 블록 지정되는데, 이걸 막아줘" |

### 1.3 반복 개선 요청 패턴

```
1차 요청: 기본 기능 구현 요청
    ↓
결과 확인 (브라우저에서)
    ↓
2차 요청: 발견한 문제점/개선사항 전달
    ↓
결과 확인
    ↓
3차 요청: 세부 조정
    ↓
완료!
```

### 1.4 핵심 팁

| 상황 | 이렇게 말하세요 |
|------|----------------|
| 위치 지정 | "~섹션 위에", "~섹션 아래에", "최상단에", "요금제 바로 앞에" |
| 기존 요소 참조 | "기존 히어로 영역은 그대로 두고", "현재 스타일을 유지하면서" |
| 구체적 콘텐츠 | 실제 텍스트, 숫자, 이름을 직접 제공 |
| 레이아웃 | "2x2 그리드", "3열 레이아웃", "가로 스크롤" |
| 인터랙션 | "드래그 가능한", "클릭하면 ~로 이동", "호버 시 확대" |

## 2. Module 1: 새로운 히어로 섹션 추가하기

### 2.1 상황

기존 랜딩페이지에 M.A.K.E.R.S AI 심사위원회 히어로가 있지만, 그 위에 정부지원금에 초점을 맞춘 새로운 히어로 섹션을 추가하고 싶다.

### 2.2 AI에게 전달할 정보

**필수 정보:**
- [ ] 메인 헤드라인 텍스트
- [ ] 서브 헤드라인 텍스트 (1~2개)
- [ ] CTA 버튼 텍스트
- [ ] 기존 히어로와의 관계 (위에 추가)

**선택 정보:**
- [ ] 원하는 색상 톤 (에메랄드, 퍼플 등)
- [ ] 강조하고 싶은 키워드
- [ ] 특별한 시각 효과 요청

### 2.3 프롬프트 예시

```
나는 현재 랜딩페이지도 좋지만, 기존의 히어로 영역이 꼭 필요했다고 생각해.
지금의 히어로 영역은 그대로 두고, 아래 내용의 히어로 영역을 최상단에 배치해줘

---
헤드라인 : 
"정부지원금 합격 사업계획서, 10분이면 충분합니다"

서브 타이틀 : 
"AI Multi-Agent가 심사위원 관점의 완벽한 초안을 제공합니다."
"예비창업패키지 · 초기창업패키지 · 정책자금지원 모두 대응"
```

### 2.4 결과 확인 체크리스트

- [ ] 새 히어로가 맨 위에 표시되는가?
- [ ] 기존 M.A.K.E.R.S 히어로는 그 아래에 그대로 있는가?
- [ ] 헤드라인/서브타이틀이 정확하게 표시되는가?
- [ ] CTA 버튼이 동작하는가?

### 2.5 추가 개선 요청 예시

> "헤드라인에서 '10분이면 충분합니다' 부분에 밑줄 애니메이션을 넣어줘"

> "CTA 버튼 아래에 '10분 완성', '합격률 극대화', '즉시 다운로드' 3개의 혜택 카드를 추가해줘"

## 3. Module 2: 고정 헤더 네비게이션 추가하기

### 3.1 상황

페이지 스크롤 시에도 항상 상단에 보이는 네비게이션 바를 추가하고 싶다.

### 3.2 AI에게 전달할 정보

**필수 정보:**
- [ ] 왼쪽 영역: 로고/브랜드명
- [ ] 중앙 영역: 이동할 섹션 목록
- [ ] 오른쪽 영역: CTA 버튼

**선택 정보:**
- [ ] 스크롤 시 스타일 변화 여부
- [ ] 모바일 메뉴 형태

### 3.3 프롬프트 예시

```
최상단에 헤드 네비게이션 바를 넣고 싶어.

네비바의 맨 왼쪽에는 메이커스라운드 로고를, 
중앙부에는 하단에 있는 주요 섹션으로 바로 이동할 수 있는 링크들을, 
맨 오른쪽에는 CTA인 사업계획서 작성 화면으로 바로 넘어갈 수 있는 버튼을 배치해줘.
```

### 3.4 결과 확인 체크리스트

- [ ] 헤더가 스크롤해도 상단에 고정되어 있는가?
- [ ] 각 링크를 클릭하면 해당 섹션으로 스크롤되는가?
- [ ] CTA 버튼이 올바른 페이지로 이동하는가?
- [ ] 스크롤 시 배경이 변하는가? (글래스모피즘)

### 3.5 추가 개선 요청 예시

> "스크롤할 때 헤더 배경에 블러 효과를 더 강하게 적용해줘"

> "모바일에서는 햄버거 메뉴로 바꿔줘"

## 4. Module 3: 섹션 순서 재배치하기

### 4.1 상황

현재 섹션 순서가 마음에 들지 않아 변경하고 싶다.

### 4.2 AI에게 전달할 정보

**필수 정보:**
- [ ] 이동할 섹션 이름
- [ ] 이동할 위치 (어떤 섹션 앞/뒤)

### 4.3 프롬프트 예시

```
문제 해결 섹션을 AI 심사위원회 섹션 앞으로 이동해줘
```

### 4.4 주의사항

섹션 순서를 변경하면 네비게이션 링크 순서도 함께 변경되어야 합니다. AI가 자동으로 처리하지만, 결과를 확인하세요.

## 5. Module 4: 사업분야 맞춤지원 섹션 추가하기

### 5.1 상황

다양한 사업 분야별로 지원 가능함을 보여주는 섹션을 추가하고 싶다.

### 5.2 AI에게 전달할 정보

**필수 정보:**
- [ ] 섹션 제목
- [ ] 서브 제목
- [ ] 각 카테고리별 정보:
  - 카테고리명
  - 설명
  - 관련 해시태그

**선택 정보:**
- [ ] 배치할 위치
- [ ] 카드 레이아웃 (2열/3열/4열)

### 5.3 프롬프트 예시

```
다음 내용을 요금제 위쪽 섹션으로 넣어서 보강해줘.
섹션 이름은 "사업분야 맞춤지원" 으로 해줘.

---
# 어떤 창업이든, 완벽하게 지원합니다

## 예비창업자부터 초기창업자까지, 모든 단계에 최적화된 솔루션

- 온라인 창업
  - 온라인 쇼핑몰, 디지털 서비스 등 온라인 기반 창업 지원
  - `#스마트스토어`, `#SaaS`

- 1인 소자본 창업
  - 최소 비용으로 시작하는 1인 창업 최적화 솔루션
  - `#프리랜서`, `#크리에이터`

- 프랜차이즈 사업
  - 프랜차이즈 사업계획서 전문 양식 지원
  - `#가맹점`, `#본사`

- AI 분야 창업
  - AI 스타트업 특화 사업계획서 및 트렌드 분석
  - `#LLM`, `#AI 에이전트`
```

## 6. Module 5: 페르소나 솔루션 섹션 추가하기

### 6.1 상황

다양한 고객 유형(페르소나)별로 어떤 요금제가 적합한지 보여주고 싶다.

### 6.2 AI에게 전달할 정보

**필수 정보 (각 페르소나별):**
- [ ] 추천 요금제 (기본/플러스/프로/프리미엄)
- [ ] 페르소나 이름과 역할
- [ ] 겪고 있는 문제 (따옴표로 인용)
- [ ] 감정 상태
- [ ] 이루고 싶은 목표

```
다음과 같은 개별 잠재 고객에 대한 솔루션으로 요금제를 뒷받침할 수 있도록

- [ ] 그룹 구분 (있는 경우)

### 7.4 결과 확인 체크리스트

- [ ] 좌우로 드래그가 가능한가?
- [ ] 각 그룹별로 리뷰가 구분되어 있는가?
- [ ] 그룹별 색상이 다른가?
- [ ] 모바일에서도 스와이프가 되는가?

## 8. Module 7: UI/UX 문제 해결하기

### 8.1 문제 발견 시 대처법

브라우저에서 결과를 확인하다가 문제를 발견하면, **증상을 구체적으로 설명**하세요.

### 8.2 일반적인 문제와 프롬프트

| 문제 상황 | 프롬프트 예시 |
|----------|--------------|
| 드래그 시 텍스트 선택됨 | "새롭게 생긴 후기 카드에 본문 영역이 텍스트 블록지정이 되어서 컴퓨터 웹에서 드래그할 때 거슬리는데, 블록지정을 막아줘" |
| 스크롤 시 헤더에 가려짐 | "섹션으로 스크롤할 때 헤더에 가려지는 부분이 있어. 고정 헤더 높이만큼 여백을 줘" |
| 모바일에서 레이아웃 깨짐 | "모바일에서 카드가 너무 작아. 모바일에서는 1열로 표시해줘" |
| 클릭이 안 됨 | "CTA 버튼을 클릭해도 아무 반응이 없어. 확인해줘" |
| 색상이 안 보임 | "배경과 텍스트 색상이 비슷해서 잘 안 보여. 대비를 높여줘" |

### 8.3 문제 설명 팁

**좋은 문제 설명:**
- 언제 발생하는지 (드래그할 때, 스크롤할 때, 클릭할 때)
- 어디서 발생하는지 (특정 섹션, 특정 버튼)
- 무엇이 문제인지 (텍스트 선택됨, 가려짐, 깨짐)
- 기대하는 동작은 무엇인지

이제 지금까지 작업한 결과물의 내용을 README.md 에 정리해줘
```

```
내가 랜딩페이지 보강 작업을 수행한 순서 또한 나중에 리뷰할 수 있도록 별도의 문서로 일목요연하게 기록해줘
```

**교육 자료:**
```
작업히스토리를 기반으로 내가 이런 작업을 다른 사람들도 할 수 있도록 교육자료를 만들고 싶어. 
docs 경로에 친절한 교육안을 작성해줘.
```
